## File-Loss Safeguard Note

Before any future destructive workflow or file-deletion action, ensure safeguards are in place: back up ignored local files such as `.env*`, confirm the blast radius with the user, and use agent hooks/guard scripts where available to block risky commands like `git clean`, `git reset --hard`, and `git switch --orphan` until the backup and confirmation happen.
