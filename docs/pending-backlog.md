  **Proposed approach (think before doing):**

  1. **Write the matrix first as a single source of truth: `docs/access-matrix.md`.** Rows = resource (table / Storage prefix / Server Action / Route Handler / page route). Columns = role (admin, manager, employee) × actor relation (self, direct report, same-department peer, stranger). Cells = allow / deny / scoped-to-X. Cover every table in `docs/database-design.md`, every action in `src/server/actions/**`, every page in `src/app/(app)/**`, every Storage prefix. Cross-reference each row to the enforcing layer (RLS policy name, action's `requireRole` call, ownership check, admin-client gate).
  2. **Make the matrix executable.** Translate each cell into a Playwright test that uses storage states for admin / manager / employee personas and asserts the documented outcome — successful read/mutation for allowed cells, redirect-to-access-denied + `auth.access_denied` audit row for denied cells, scoped-to-N rows for scoped cells (e.g. manager seeing only direct-report leave). Include direct-DB / direct-Storage / forged-UUID forms, not only UI flows, because UI hides don't equal denied access.
  3. **Tie new code to the matrix in CI.** Any PR touching `src/server/actions/**`, a new `src/app/(app)/**` route, a new table/RLS migration, or a Storage policy must update `docs/access-matrix.md` and add the matching test row. Enforce with a lightweight `tools/check-access-matrix.mjs` script in `npm run check` that fails if a new action/route/table/policy is not represented. (Doc-as-test-spec pattern; same principle as `docs/rls-policy-map.md` but at the application boundary.)
  4. **Cross-check the two matrices.** `docs/rls-policy-map.md` covers the DB layer; the new `docs/access-matrix.md` covers the application layer. Inconsistencies (DB allows, app denies — or vice versa) are bugs.
  5. **Negative-path audit assertions are mandatory.** Every denied cell must produce an `audit_logs` row with `action = 'auth.access_denied'` (`docs/systems-thinking.md` §2 feedback rule). Test fails if either the deny or the audit row is missing.
  6. **Run on every PR.** The matrix suite stays green or merge blocks. Treat a red as P0.
